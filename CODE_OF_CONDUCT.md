# Code of Conduct

## Purpose

This project aims to build a professional, respectful, and collaborative environment for all contributors.  
By participating in this project, all team members agree to maintain a safe, constructive, and respectful space for communication, development, review, and teamwork.

---

## Our Standards

All contributors are expected to:

- Communicate respectfully with all team members
- Accept feedback professionally and respond constructively
- Focus on solving problems instead of creating conflict
- Respect deadlines, responsibilities, and shared decisions
- Write clean, understandable, and maintainable code
- Follow the agreed project structure, naming conventions, and workflow
- Collaborate honestly and avoid taking credit for work not completed by them
- Help maintain a supportive environment for learning and teamwork

---

## Unacceptable Behavior

The following behaviors are not acceptable in this project:

- Insulting, offensive, or disrespectful language
- Harassment, bullying, intimidation, or personal attacks
- Repeatedly ignoring agreed project rules or team decisions
- Damaging, deleting, or modifying teammates’ work without coordination
- Pushing unreviewed or unrelated code directly in a way that disrupts the project
- Sharing sensitive project information, credentials, or private data without permission
- Plagiarism or submitting copied work without clear acknowledgment
- Creating unnecessary conflict, blame, or hostility within the team

---

## Collaboration Rules

To keep the project organized, all contributors must follow these collaboration rules:

- Each task should be completed within its assigned feature branch
- Changes should be submitted through Pull Requests
- Team members should avoid working directly on unrelated features
- Shared files and common structures must be edited carefully and with coordination
- Contributors must test their work before submitting it for review
- Commit messages and Pull Requests should be clear and relevant to the work done

---

## Code Quality Expectations

All contributors are expected to:

- Keep code readable and organized
- Reuse shared components when possible
- Avoid unnecessary duplication
- Follow the agreed architecture and folder structure
- Keep business logic separated from UI
- Add comments only when they provide real value
- Ensure that their code does not break existing functionality

---

## Respect for Team Workflow

This project uses a structured workflow based on feature tasks and feature branches.  
All contributors are expected to respect:

- assigned responsibilities
- branch naming conventions
- review process
- project deadlines
- technical decisions agreed upon by the team

Disagreements should be discussed professionally and resolved based on logic, project needs, and team consensus.

---

## Reporting Issues

If any team member experiences behavior that violates this Code of Conduct, they should report it to the project lead or team supervisor privately and clearly.

All reports should be handled fairly, respectfully, and with appropriate confidentiality.

---

## Enforcement

Project maintainers or team leads are responsible for enforcing this Code of Conduct.  
They may take appropriate action when rules are violated, including:

- giving a warning
- requesting changes in behavior
- restricting contribution access
- removing a contributor from the project if necessary

---

## Scope

This Code of Conduct applies to all project-related spaces, including:

- source code contributions
- pull requests
- code reviews
- project discussions
- team chats
- meetings
- documentation
- issue tracking tools

---

## Final Note

This project is a shared responsibility.  
Its success depends not only on technical skills, but also on discipline, respect, accountability, and teamwork.
